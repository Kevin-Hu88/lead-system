# 数字营销系统 - 项目状态文档

> 本文档记录系统当前架构、已完成改造、待办事项。
> 新对话/新电脑接手时，先读此文件即可快速了解全貌。

## 系统概述

膜结构车棚/遮阳棚/光伏车棚厂家的获客+营销自动化系统。

**核心功能：**
- 多渠道线索采集（百度地图、招标平台、在建工程、搜索引擎、58同城等）
- 自动评分分级（S/A/B/C 四级）
- 分级自动触达（S 级人工跟进、A/B 级自动短信、C 级跳过）
- 数据质量评估（需求信号加权、冷数据降级）
- Web 仪表盘（线索管理、采集控制、竞品分析）

## 技术栈

- Python 3.9+ / Flask
- SQLite (crm.db)
- APScheduler 定时任务
- Selenium 招标采集
- Jinja2 模板

### 4. 数据质量
- [x] 百度地图 POI 自动降级为 C 级（无需求信号时）
- [x] 需求信号加权（招标/在建/改造 → 自动升级 S/A）
- [x] 自动触达只走 B 级以上，C 级跳过
- [x] 冷数据自动标记 `[冷数据]`
- [x] **地点类型 vs 实体类型区分**（2026-06-08）
    - 地点类型（工业园/停车场/物流园/充电站等）= 非决策主体，评分大幅压低
    - 实体类型（工厂/4S店/学校/医院等）= 真正决策主体，正常评分
    - 百度地图 + 地点类型 + 无信号 → 强制 C 级，封顶 12 分
    - 移除了地点关键词对 SS/A 级的误触发
    - 只对百度地图 POI 做地点类型检测，避免误判经营范围中包含地点词的企业
- [x] **产品兴趣词不再作为需求信号**（2026-06-08）
    - product_interest 是采集器搜索关键词，不是客户需求
    - 只检查 demand_desc 和 notes 中的信号
